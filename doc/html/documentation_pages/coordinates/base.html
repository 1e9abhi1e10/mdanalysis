4.1. Base classes — MDAnalysis.coordinates.base¶

Derive other Reader and Writer classes from the classes in this module.

class MDAnalysis.coordinates.base.Timestep(arg)¶

Timestep data for one frame

Data: numatoms - number of atoms
frame - frame number dimensions - system box dimensions (x, y, z, alpha, beta, gamma)
Methods: t = Timestep(numatoms) - create a timestep object with space for numatoms (done automatically)
t[i] - return coordinates for the i’th atom (0-based) t[start:stop:skip] - return an array of coordinates, where start, stop and skip correspond to atom indices (0-based)
dimensions¶

get unitcell dimensions (A, B, C, alpha, beta, gamma)

class MDAnalysis.coordinates.base.IObase¶

Base class bundling common functionality for trajectory I/O.

close()¶

Close the trajectory file.

close_trajectory()¶

Specific implementation of trajectory closing.

convert_pos_from_native(x)¶

In-place conversion of coordinate array x from native units to base units.

convert_pos_to_native(x)¶

In-place conversion of coordinate array x from base units to native units.

convert_time_from_native(t)¶

Convert time t from native units to base units.

convert_time_to_native(t)¶

Convert time t from base units to native units.

format¶

override to define trajectory format of the reader/writer (DCD, XTC, ...)

units¶

dict with units of of time and length (and velocity, force, ... for formats that support it)

class MDAnalysis.coordinates.base.Reader¶

Base class for trajectory readers.

See Trajectory API definition in MDAnalysis.coordinates for the required attributes and methods.

dt¶

Time between two trajectory frames in picoseconds, rounded to 4 decimals.

next()¶

Forward one step to next frame.

rewind()¶

Position at beginning of trajectory

totaltime¶

Total length of the trajectory numframes * dt.

class MDAnalysis.coordinates.base.Writer¶

Base class for trajectory writers.

See Trajectory API definition in MDAnalysis.coordinates for the required attributes and methods.

convert_dimensions_to_unitcell(ts)¶

Read dimensions from timestep ts and return appropriate unitcell

write(obj)¶

Write current timestep, using the supplied obj.

The argument should be a AtomGroup or a Universe or a Timestep instance.

Note

The size of the obj must be the same as the number of atom provided when setting up the trajectory.
